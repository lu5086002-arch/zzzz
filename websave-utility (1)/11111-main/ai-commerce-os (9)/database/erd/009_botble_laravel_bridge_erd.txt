2. Cardinality & Relationship Constraints
================================================================================

1. botble_sync_configurations (1) <----> (1) (tenant_id, store_id bound)
   - Guarantees exactly one endpoint mapping per unique tenant+store pair.

2. botble_webhook_events (1) <----> (0..N) botble_collab_synergy_runs
   - Each verified webhook event can trigger an active multi-agent synergy run thread
     aimed at resolving specific operational challenges (e.g. bulk procurement, SEO optimization, CRM email queues).

3. botble_collab_synergy_runs (1) <----> (0..N) botble_agent_tasks
   - A single collaborative synergy run can result in zero or many secure, audited action commands
     directed back to the host Botble CMS (or stored in the system's ledger upon passing security criteria).

4. botble_agent_tasks (1) <----> (1) ECOS task_requests
   - Maps 1-to-1 against the core tenant-isolated safety and limit rules enforcer.
