# EPIC 2: 🍔 餐馆外卖 – 客单价提升 & 复购智能体 (AOV & Retention Agent)

## 1. 背景 & 业务痛点
**行业**：餐馆外卖 (restaurant_takeout)

**典型痛点**：
- **客单价低**：客人倾向于只点主食，导致单均收益覆盖不了高昂的平台抽佣。
- **毛利吃紧**：缺乏科学的套餐定价，盲目打折导致亏损。
- **复购率不稳**：新客留存难，缺乏系统化的「首单→二单」唤醒流程。
- **运营盲区**：不知道哪些菜适合搭在一起卖，也不知道优惠券发给谁最有效。

**解决方案**：
通过 AI Commander 自动分析菜品动销结构，设计高毛利套餐，并针对新老客建立自动化的复购营销流（Flow）。

---

## 2. 业务目标 (Goal) 示例
**自然语言输入**：
- 「帮我提升外卖客单价 15%。」
- 「让最近一个月的新客更多人回来点第二单。」

**结构化 BusinessGoal**：
```typescript
const goal: BusinessGoal = {
  timeRange: {
    preset: 'next_month',
  },
  metricsTarget: {
    gmvChangeRate: 0.15, // 目标：GMV 提升 15%
  },
  priorityWeights: {
    gmv: 0.25,
    margin: 0.20,
    inventoryHealth: 0.10,
    retention: 0.30, // 复购是餐饮的核心
    risk: 0.15,      // 关注差评/退款风险
  },
};
```

## 3. Ops Commander 的「增长 Playbook」结构
**主目标**：在下个月，通过套餐设计与复购策略，提升全店 AOV (平均客单价) 和新客复购率。

### 模块 A：智能套餐设计 (客单价提升)
- **策略**：分析畅销主菜与高毛利小食/饮料的搭配率，设计「1+1+1」套餐。
- **定价建议**：套餐价比单买总价低 10-15%，确保覆盖成本并吸引下单。
- **输出**：新套餐 SKU 草稿及平台推广位建议。

### 模块 B：新客复购自动化流 (Retention Flow)
- **策略**：针对首单客户，设计「感谢→发券→唤醒」三阶段自动提醒。
- **执行**：
  - T+0: 下单后发送专业感谢语。
  - T+3: 发送 15% off 二单专属券。
  - T+10: 针对未复购者，基于首单口味推荐相关新菜。
- **输出**：营销 Flow 草稿。

### 模块 C：出品质量与差评风控
- **策略**：识别近 30 天高频差评菜品或时段。
- **行动**：建议优化菜品口味或在高峰期限制单量以保质量。
- **输出**：服务质量优化建议清单。

## 4. 需要调用的智能体 & 工具链路
### 4.1 涉及角色
- **Ops Commander**：负责增长逻辑闭环与任务指派。
- **Pricing Agent**：负责菜品成本理算与套餐定价。
- **Marketing Agent**：负责活动策划、文案生成及营销流配置。
- **Risk Agent**：负责差评分析与出品风险预警。

### 4.2 工具链路 (Logic Flow)
1. **分析阶段**：调用 `ProductService.getProductInsight` 和 `MenuService.getTopItems`。
   - 获取主食、小食、饮料的销量排名与毛利数据。
2. **定价阶段**：`Pricing Agent` 结合成本数据，生成套餐组合。
3. **营销阶段**：`Marketing Agent` 调用 `CustomerService.getRecentFirstOrderCustomers`。
   - 匹配目标人群并生成 Flow 配置草稿。
4. **风控阶段**：`Risk Agent` 调用 `ReviewService.getRecentBadReviews` 分析痛点。

## 5. 用户故事 (User Stories)
- **Story 1 (店长)**：作为餐厅店长，我希望系统自动组合出最受欢迎的套餐，以便在不需要人工干预的情况下提升客单价。
- **Story 2 (运营)**：作为外卖运营，我希望有一套自动化的发券流程来留住新客户，以便提升店铺的复购率。
- **Story 3 (品质控)**：作为厨师长，我希望知道哪些菜品导致了差评，以便针对性地改进烹饪流程。

---

## 6. 验收标准 (Acceptance Criteria)
1. 能基于真实/模拟销量数据，自动识别出「主食+小食+饮料」的最优组合方案。
2. 针对「提升复购」目标，能输出完整的自动化营销流草稿（触发条件、文案、优惠力度）。
3. 针对差评数据，能准确归因到具体的菜品或时间段，并给出整改建议。
4. 所有建议均需以「结构化草稿」形式呈现，支持用户一键确认部署。
