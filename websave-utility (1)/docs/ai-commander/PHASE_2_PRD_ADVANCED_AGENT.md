# 高级运营智能体 v2 PRD 大纲：从单体问答到多角色目标驱动

## 1. 项目愿景
将 AI Commander 从“响应式顾问”升级为“主动式副总裁”。通过多角色分工、目标驱动规划和行为记忆复盘，构建一套完整的运营智能体体系。

## 2. 新增核心能力

### 2.1 多智能体角色分工 (Multi-Agent Roles)
在调度层逻辑上拆分出 5 个专业角色，每个角色拥有独立的 System Prompt 和工具集：
- **运营总监 (Ops Commander)**：负责全局目标、优先级排序及任务指派。
- **定价与商品专家 (Pricing & Merch Agent)**：专注定价策略、单品洞察与促销方案。
- **库存专家 (Inventory Agent)**：监控库存健康、缺货预警与积压风险。
- **支付专家 (Payments Agent)**：优化结账转化率与支付组合。
- **风控专家 (Risk Agent)**：负责订单安全评估与欺诈拦截。

### 2.2 目标驱动规划 (Goal-driven Planning)
支持用户下达数值化目标，AI 自动拆解行动步骤：
- **示例指令**：“帮我下个月 GMV 提升 20%”。
- **内部流程**：目标识别 -> 指标拆解 -> 诊断现状 -> 编制战术包 (Playbook) -> 输出操作草稿。

### 2.3 行为记忆与复盘 (Memory & Reflection)
- **AI 行为日志 (ai_actions_log)**：记录所有 AI 提议并被执行的动作。
- **闭环分析**：支持提问“上次你建议的活动效果如何？”，AI 自动对比执行前后的业务数据并给出复盘结论。

### 2.4 动作草稿生成 (Actionable Drafts)
- **不再只说空话**：建议促销时，同步生成折扣 JSON 草稿；建议补货时，生成采购单草稿。
- **人类确认机制**：保留“人类最后一击”，点击确认按钮才正式写入生产数据库。

## 3. 技术路线图
1. **角色映射 (Sprint 4)**：实现 Intent -> Agent Role 的自动切换逻辑。
2. **动作型工具 (Sprint 5)**：开发 Discount, Flow, Pricing 等草稿生成接口。
3. **行为日志 (Sprint 6)**：建立 `ai_actions_log` 存储与回溯分析能力。
4. **目标拆解 (Sprint 7)**：实现 Planning 引擎，支持复杂目标任务。

## 4. 安全边界
- **只读默认**：除明确定义的草稿生成外，AI 无权修改任何生产配置。
- **可追溯性**：每一条 AI 建议都必须标记来源角色和依据的数据指标。
